## Architecture

This is a **Stellar Scaffold** frontend — a Vite + SvelteKit + TypeScript app for building Stellar/Soroban smart contract UIs. Uses `adapter-static` with `fallback: 'index.html'` for SPA mode (no server required).

### Key flows

**Contract client generation**: `stellar scaffold watch --build-clients` (part of `npm run dev`) watches for contract changes, compiles Rust contracts to WASM, deploys them, and generates TypeScript clients into `packages/<contract-name>/`. These packages are npm workspace members and are re-imported as `src/lib/contracts/<contract_name>.ts`.

**Stub files**: `src/lib/contracts/*.ts` (except `util.ts`) are stub files that exist so the build succeeds before contracts are generated. When the real client is generated, it replaces these stubs. All contract files export a `default` instance of the contract client and import `rpcUrl`/`networkPassphrase` from `./util`.

**Environment config**: Two separate config systems:
- `.env` (copy from `.env.example`): Runtime env vars for the frontend, prefixed with `PUBLIC_` to be exposed by Vite
- `scaffold.yml`: Stellar Scaffold CLI config defining networks, accounts, and contracts per environment

### Important directories

- **`src/lib/contracts/`**: Contract client files auto-generated by Stellar Scaffold CLI. `util.ts` reads env vars with Zod validation and exports `rpcUrl`, `networkPassphrase`, `network`, and `labPrefix()`.
- **`src/lib/stores/wallet.ts`**: Svelte writable stores for wallet state. Module-level polling starts on import. Exports `address`, `balances`, `network`, `networkPassphrase`, `isPending`, `walletWarnings`, `signTransaction`, `updateBalances()`.
- **`src/lib/stores/notifications.ts`**: `writable<Notification[]>` store with `addNotification(message, type)`.
- **`src/lib/util/wallet.ts`**: `StellarWalletsKit` instance, `connectWallet`/`disconnectWallet`, `fetchBalances` via Horizon.
- **`src/lib/components/`**: Svelte components — `ConnectAccount`, `WalletButton`, `NetworkPill`, `FundAccountButton`, `GuessTheNumber`.
- **`packages/`**: npm workspace packages for generated contract TypeScript bindings (populated by `stellar scaffold`).
- **`contracts/`**: Rust/Soroban smart contract source (Cargo workspace with `contracts/*` members).

### Routes
- `/` — Home page (`src/routes/+page.svelte`)
- `/debug` — Contract Explorer placeholder (`src/routes/debug/+page.svelte`)

### SvelteKit notes
- `src/routes/+layout.ts` sets `prerender = true` and `ssr = false` for static SPA mode
- Svelte 5 runes mode enabled for all non-`node_modules` files via `svelte.config.js`
- Use `$state()`, `$derived()`, `$effect()` for reactivity; `$props()` for component props
- Wallet and notification state use Svelte stores (`writable`, `derived`) from `svelte/store`
- Store auto-subscription syntax (`$storeName`) works in `.svelte` files

### Vite configuration notes
- Env vars must be prefixed with `PUBLIC_` to be exposed in frontend code
- WASM support enabled via `vite-plugin-wasm`
- Node `Buffer` polyfill provided via `vite-plugin-node-polyfills`
- `@stellar/stellar-xdr-json` excluded from dep optimization (uses WASM)
- Local friendbot proxied at `/friendbot` → `http://localhost:8000/friendbot`
