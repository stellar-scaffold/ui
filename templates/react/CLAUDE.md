## Commands

```bash
# Start dev server (runs stellar scaffold watch + vite concurrently)
npm run dev

# Build for production
npm run build

# Lint
npm run lint

# Format

# Build contract TypeScript clients (after contracts are deployed)
npm run install:contracts   # install deps + build
npm run build:contracts     # build only
```

There are no automated tests in this project.

## Architecture

This is a **Scaffold Stellar** frontend — a Vite + React + TypeScript app for
building Stellar/Soroban smart contract UIs.

### Key flows

**Contract client generation**: `stellar scaffold watch --build-clients` (part
of `npm run dev`) watches for contract changes, compiles Rust contracts to WASM,
deploys them, and generates TypeScript clients into `packages/<contract-name>/`.
These packages are npm workspace members and are re-imported as
`src/contracts/<contract_name>.ts`.

**Stub files**: `src/contracts/*.ts` (except `util.ts`) are stub files that
exist so the build succeeds before contracts are generated. When the real client
is generated, it replaces these stubs. All contract files export a `default`
instance of the contract client and import `rpcUrl`/`networkPassphrase` from
`./util`.

**Environment config**: Two separate config systems:

- `.env` (copy from `.env.example`): Runtime env vars for the frontend, prefixed
  with `PUBLIC_` to be exposed by Vite
- `environments.toml`: Scaffold Stellar CLI config defining networks, accounts,
  and contracts per environment (`development`, `staging`, `production`)

### Provider tree (`src/main.tsx`)

`NotificationProvider` → `QueryClientProvider` → `WalletProvider` →
`BrowserRouter`

### Important directories

- **`src/contracts/`**: Contract client files auto-generated by Scaffold Stellar
  CLI. `util.ts` reads env vars with Zod validation and exports `rpcUrl`,
  `networkPassphrase`, `network`, and `labPrefix()`.
- **`src/debug/`**: Full-featured contract debugger UI at
  `/debug/:contractName`. Uses `import.meta.glob("../../contracts/*.ts")` to
  dynamically load all contract clients. Renders contract methods as interactive
  forms using JSON Schema from contract metadata.
- **`src/providers/WalletProvider.tsx`**: Manages wallet connection state via
  Stellar-Wallets-Kit v2 state events (`onWalletStateChange` /
  `onWalletDisconnect` from `@stellar-scaffold/app-lib`). No polling — the kit
  owns persistence and restores on reload. Exposes `address`, `balances`,
  `networkPassphrase`, `isPending`, `signTransaction`, `updateBalances`.
- **`src/hooks/useWallet.ts`**: Consume `WalletContext` in components.
- Wallet primitives (`StellarWalletsKit` init, `connectWallet`,
  `disconnectWallet`, `profileModal`, `signTransaction`, `fetchBalances`) live
  in the shared `@stellar-scaffold/app-lib` package, not in the template. v2
  ships its own connect/profile modals, so the template no longer renders a
  custom disconnect dialog or per-wallet warning badges.
- **`packages/`**: npm workspace packages for generated contract TypeScript
  bindings (populated by `stellar scaffold`).
- **`contracts/`**: Rust/Soroban smart contract source (Cargo workspace with
  `contracts/*` members).

### Routes

- `/` — Home page (`src/pages/Home.tsx`)
- `/debug` and `/debug/:contractName` — Contract debugger
  (`src/pages/Debugger.tsx`)

### Vite configuration notes

- Env vars must be prefixed with `PUBLIC_` to be exposed in frontend code
- WASM support enabled via `vite-plugin-wasm`
- Node `Buffer` polyfill provided via `vite-plugin-node-polyfills`
- `@stellar/stellar-xdr-json` excluded from dep optimization (uses WASM)
- Local friendbot proxied at `/friendbot` → `http://localhost:8000/friendbot`
